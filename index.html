Four Pillars
✓
HIGH What is OOPs? Explain the four pillars. ›
Best Answer
OOP (Object-Oriented Programming) is a paradigm that models software around real-world objects. The four pillars are:

1. Encapsulation — Bundling data (fields) and methods in one unit (class), hiding internal state via access modifiers. Example: ATM machine — you press buttons but don't see internal circuits.

2. Abstraction — Hiding implementation details, showing only necessary features. Example: Car's steering wheel — you turn it, you don't need to know how the rack-and-pinion works.

3. Inheritance — A child class acquires properties/methods of a parent class using extends keyword. Promotes code reuse. Example: Dog extends Animal → Dog inherits eat(), breathe() from Animal.

4. Polymorphism — Same method/operator behaving differently in different contexts. Two types: Compile-time (method overloading) and Runtime (method overriding).
💡 Always give a real-world analogy. Interviewers love it when you connect concepts to everyday objects.
✓
HIGH Difference between method overloading and method overriding. ›
Best Answer
Overloading (Compile-time Polymorphism) — Same method name, different parameters, in the same class.
Overriding (Runtime Polymorphism) — Child class provides its own implementation of a parent class method with the same signature.
// Overloading class Calculator { int add(int a, int b) { return a + b; } double add(double a, double b) { return a + b; } // same name, diff params } // Overriding class Animal { void sound() { System.out.println("..."); } } class Dog extends Animal { @Override void sound() { System.out.println("Woof!"); } // same signature, diff body }
✓
HIGH Difference between interface and abstract class. ›
Best Answer
Abstract Class: Can have abstract + concrete methods, constructors, instance variables. A class can extend only ONE abstract class.

Interface: All methods are abstract by default (before Java 8). No constructors, no instance variables (only constants). A class can implement MULTIPLE interfaces.

When to use: Use abstract class for "is-a" relationship with shared code. Use interface for "can-do" capability contracts.
abstract class Vehicle { abstract void start(); void stop() { /*...*/ } } interface Flyable { void fly(); } class FlyingCar extends Vehicle implements Flyable { void start() { System.out.println("Starting"); } public void fly() { System.out.println("Flying!"); } }
✓
HIGH What is the final keyword in Java? ›
Best Answer
final can be applied to three things:

Variable: Value cannot be changed (constant). final int X = 10;
Method: Cannot be overridden in subclass.
Class: Cannot be extended (inherited). Example: String class is final in Java.
✓
HIGH final vs finally vs finalize in Java ›
Best Answer
final — keyword for constants, non-overridable methods, non-inheritable classes.
finally — block in try-catch that ALWAYS executes, used for cleanup (closing files/DB connections).
finalize() — method called by GC before an object is garbage collected. Deprecated in Java 9+.
✓
HIGH What is exception handling? Explain with code. ›
Best Answer
Exception handling is a mechanism to handle runtime errors gracefully, maintaining normal program flow. Key keywords: try, catch, finally, throw, throws.
try { int result = 10 / 0; // ArithmeticException } catch (ArithmeticException e) { System.out.println("Cannot divide by zero: " + e.getMessage()); } finally { System.out.println("This always runs — cleanup code here"); }
Checked exceptions (must handle) vs Unchecked exceptions (runtime, optional to handle). Custom exception: extend Exception class.
✓
MEDIUM What is Java garbage collection? ›
Best Answer
Garbage Collection (GC) is the automatic process of reclaiming heap memory used by objects that are no longer referenced. Java uses Mark and Sweep algorithm. The JVM's GC runs automatically — you can suggest it with System.gc() but cannot force it.

Why Java is platform independent: Java code compiles to bytecode (.class), which runs on any JVM regardless of OS — "Write Once, Run Anywhere."
✓
MEDIUM What is synchronization in Java? What is deadlock? ›
Best Answer
Synchronization ensures only one thread accesses a shared resource at a time using the synchronized keyword. Prevents race conditions in multithreading.

Deadlock occurs when two or more threads are blocked forever, each waiting for a resource held by the other.
Example: Thread A holds Lock 1, waits for Lock 2. Thread B holds Lock 2, waits for Lock 1 → deadlock.

Prevention: Always acquire locks in a fixed order, use timeout, or use tryLock().
✓
MEDIUM String vs StringBuilder vs StringBuffer ›
Best Answer
String: Immutable. Every modification creates a new object. Stored in String Pool.
StringBuilder: Mutable. NOT thread-safe. Faster. Use in single-threaded code.
StringBuffer: Mutable. Thread-safe (synchronized methods). Slightly slower.

Rule of thumb: Use String for constants, StringBuilder for string manipulation in single threads, StringBuffer in multithreaded scenarios.
✓
MEDIUM What are Java 8 key features? ›
Best Answer
Key Java 8 additions: Lambda Expressions (anonymous functions), Stream API (functional-style operations on collections), Optional class (null safety), Default methods in interfaces, Date/Time API (java.time), Functional Interfaces.
// Lambda List<String> names = Arrays.asList("Alice","Bob","Charlie"); names.forEach(name -> System.out.println(name)); // Stream int sum = IntStream.rangeClosed(1, 10).sum(); // 55

HIGH Mutable vs immutable data types in Python ›
Best Answer
Mutable (can be changed after creation): list, dict, set, bytearray.
Immutable (cannot be changed): int, float, str, tuple, bool, frozenset.

Example: a = "hello" → you can't change a character in place; you create a new string. But my_list = [1,2,3]; my_list.append(4) — modifies in-place.

Best Answer

LOW How does Python manage memory? ›
Best Answer
Python uses reference counting as its primary GC mechanism. When an object's reference count drops to 0, memory is freed. Python also has a cyclic GC to detect reference cycles. The memory allocator (pymalloc) manages small objects. Everything in Python is an object stored on the heap; the stack holds references.

HIGH Binary search algorithm — write code ›
Best Answer
Binary search works on sorted arrays. Time complexity: O(log n).
def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1 # not found
✓
HIGH Fibonacci series — iterative AND recursive ›
Best Answer
// Iterative (preferred - O(n) time, O(1) space) void fibonacci(int n) { int a = 0, b = 1; for (int i = 0; i < n; i++) { System.out.print(a + " "); int c = a + b; a = b; b = c; } } // Recursive (O(2^n) — mention the inefficiency!) int fib(int n) { if (n <= 1) return n; return fib(n-1) + fib(n-2); }
💡 TCS asked: write iterative FIRST, then asked to convert to recursive. Know both!

✓
MEDIUM Bubble sort and Insertion sort — write code + explain ›
Best Answer
def bubble_sort(arr): n = len(arr) for i in range(n): for j in range(0, n-i-1): if arr[j] > arr[j+1]: arr[j], arr[j+1] = arr[j+1], arr[j] # Time: O(n²), Space: O(1) def insertion_sort(arr): for i in range(1, len(arr)): key = arr[i] j = i - 1 while j >= 0 and arr[j] > key: arr[j+1] = arr[j]; j -= 1 arr[j+1] = key # Good for nearly-sorted arrays
✓
MEDIUM Find middle element of a linked list (slow-fast pointer) ›
Best Answer
Use the two-pointer technique: slow moves 1 step, fast moves 2 steps. When fast reaches end, slow is at middle.
ListNode findMiddle(ListNode head) { ListNode slow = head, fast = head; while (fast != null && fast.next != null) { slow = slow.next; fast = fast.next.next; } return slow; }
✓
MEDIUM What is a Binary Search Tree? Traversals? ›
Best Answer
BST: left child < node < right child. Allows O(log n) search in balanced BST.

Inorder (LNR): gives sorted order. Preorder (NLR): root first. Postorder (LRN): root last. BFS: level by level using queue.
✓
MEDIUM Explain time complexity with example ›
Best Answer
Time complexity measures how runtime grows with input size n.
O(1): array access — constant
O(log n): binary search — halving
O(n): linear search — one pass
O(n log n): merge sort, quick sort
O(n²): bubble sort, nested loops
O(2^n): recursive fibonacci

SQL & DBMS
SQL queries are asked in ~90% of interviews. Write them — don't just explain.
0/0 done
✓
HIGH DELETE vs DROP vs TRUNCATE ›
Best Answer
DELETE: DML — removes specific rows (with WHERE clause). Transaction log kept, rollback possible.
TRUNCATE: DDL — removes ALL rows, faster (no row-level logs), no WHERE clause, cannot rollback in most DBs.
DROP: DDL — deletes the entire table structure AND data. Irreversible.
DELETE FROM employees WHERE id = 5; -- removes 1 row TRUNCATE TABLE employees; -- removes all rows, table remains DROP TABLE employees; -- table gone completely
✓
HIGH ACID properties in DBMS ›
Best Answer
A — Atomicity: Transaction is all-or-nothing. Either all operations succeed or none do. (Bank transfer: debit + credit must both happen.)
C — Consistency: Database moves from one valid state to another. Constraints are always maintained.
I — Isolation: Concurrent transactions don't interfere with each other.
D — Durability: Once committed, data persists even after crashes.
✓
HIGH Types of SQL Joins — explain all ›
Best Answer
INNER JOIN: Returns matching rows in BOTH tables.
LEFT JOIN: All rows from left + matching from right (NULLs if no match on right).
RIGHT JOIN: All rows from right + matching from left.
FULL OUTER JOIN: All rows from both tables.
SELF JOIN: Table joined with itself.
CROSS JOIN: Cartesian product of both tables.
-- Left Join Example SELECT e.name, d.department FROM employees e LEFT JOIN departments d ON e.dept_id = d.id; -- Self Join (find employees in same department) SELECT a.name, b.name FROM emp a, emp b WHERE a.dept = b.dept AND a.id != b.id;
✓
HIGH Find person with maximum salary — write SQL query ›
Best Answer
-- Method 1: Subquery SELECT * FROM employees WHERE salary = (SELECT MAX(salary) FROM employees); -- Method 2: ORDER BY + LIMIT SELECT * FROM employees ORDER BY salary DESC LIMIT 1;
✓
HIGH Find 2nd highest salary WITHOUT OFFSET/LIMIT ›
Best Answer
-- Using subquery (most elegant) SELECT MAX(salary) FROM employees WHERE salary < (SELECT MAX(salary) FROM employees); -- Using DENSE_RANK SELECT salary FROM ( SELECT salary, DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk FROM employees ) t WHERE rnk = 2;
💡 This exact query was asked in multiple real TCS interviews. Know both methods!
✓
HIGH RANK vs DENSE_RANK ›
Best Answer
RANK(): Gives same rank to ties, but SKIPS the next rank. (1, 1, 3, 4…)
DENSE_RANK(): Gives same rank to ties, NO gaps. (1, 1, 2, 3…)
SELECT name, salary, RANK() OVER (ORDER BY salary DESC) AS rank, DENSE_RANK() OVER (ORDER BY salary DESC) AS dense_rank FROM employees;
✓
HIGH Count duplicate email IDs in a table — write SQL ›
Best Answer
SELECT email, COUNT(*) AS occurrences FROM users GROUP BY email HAVING COUNT(*) > 1 ORDER BY occurrences DESC;
✓
HIGH What is indexing? How does it reduce query time? ›
Best Answer
An index is a data structure (usually B-tree) that speeds up data retrieval. Like an index in a book — instead of scanning every page, you jump to the right one.

For slow JOINs: Add index on the JOIN column. Example: if joining on emp.dept_id, index that column.
Trade-off: Indexes speed up reads but slow down writes (INSERT/UPDATE/DELETE) since the index must also be updated.
CREATE INDEX idx_dept ON employees(dept_id);
✓
MEDIUM SQL Views — what are they and why use them? ›
Best Answer
A view is a virtual table based on a SELECT query. It doesn't store data; it's a saved query. Benefits: security (hide sensitive columns), simplify complex queries, present data in a specific format.
CREATE VIEW high_earners AS SELECT name, salary FROM employees WHERE salary > 100000; SELECT * FROM high_earners; -- use like a table
✓
MEDIUM Database vs Data Warehousing ›
Best Answer
Database (OLTP): Optimized for day-to-day transactional operations (INSERT/UPDATE/DELETE). Small queries, high concurrency. Example: bank transactions.

Data Warehouse (OLAP): Optimized for analytical queries and reporting. Large historical datasets, complex aggregations. Example: Snowflake, Amazon Redshift — for business intelligence reports.
✓
MEDIUM Types of SQL keys (Primary, Foreign, Unique, Candidate, Composite) ›
Best Answer
Primary Key: Uniquely identifies each row. NOT NULL + UNIQUE. One per table.
Foreign Key: References Primary Key of another table. Enforces referential integrity.
Unique Key: Ensures unique values, allows one NULL.
Candidate Key: Any column set that could serve as primary key.
Composite Key: Primary key made of multiple columns.
✓
MEDIUM Normalization — explain BCNF and 4NF ›
Best Answer
1NF: Atomic values, no repeating groups.
2NF: 1NF + no partial dependency.
3NF: 2NF + no transitive dependency.
BCNF: Stronger 3NF — every determinant must be a super key.
4NF: BCNF + no multi-valued dependencies.

Goal of normalization: eliminate redundancy and update anomalies.

Computer Networks
Asked selectively — mainly if mentioned in resume or specific panels
0/0 done
✓
HIGH OSI model vs TCP/IP model — all layers ›
Best Answer
OSI (7 layers): Physical, Data Link, Network, Transport, Session, Presentation, Application. Mnemonic: "Please Do Not Throw Sausage Pizza Away"

TCP/IP (4 layers): Network Access, Internet, Transport, Application.

OSI is a reference model (conceptual). TCP/IP is the actual implemented model used on the internet.
✓
MEDIUM What is subnetting? Why is it used? ›
Best Answer
Subnetting divides a large network into smaller sub-networks (subnets) using a subnet mask. Benefits: better IP address management, reduced broadcast traffic, improved security.

Example: 192.168.1.0/24 — the /24 means 24 bits for network, 8 for hosts = 256 host addresses.
✓
MEDIUM Difference between masking and encryption ›
Best Answer
Masking: Replaces sensitive data with fake but realistic data. Original data is irretrievably hidden. Used for testing/dev environments. Not reversible.
Encryption: Transforms data using an algorithm + key. Reversible — original data can be recovered with the correct key. Used for secure transmission.

Web Tech & APIs

✓
HIGH What are REST APIs? RESTful principles. ›
Best Answer
REST (Representational State Transfer) is an architectural style for APIs. RESTful APIs use HTTP methods to perform CRUD:

GET → Read | POST → Create | PUT/PATCH → Update | DELETE → Delete

PATCH vs PUT: PUT replaces the entire resource. PATCH partially updates specific fields.

Principles: Stateless, Client-Server, Uniform Interface, Cacheable, Layered System.
✓
HIGH What is CRUD? Explain with SQL and HTTP mapping. ›
Best Answer
CRUD = Create, Read, Update, Delete — the four basic operations on data.
SQL: INSERT / SELECT / UPDATE / DELETE
HTTP: POST / GET / PUT-PATCH / DELETE
✓
HIGH Git vs GitHub — what is the difference? ›
Best Answer
Git is a distributed version control system (a tool on your computer) for tracking code changes.
GitHub is a cloud hosting service for Git repositories + collaboration features (PRs, Issues, Actions).

Analogy: Git is the camera; GitHub is Instagram where you share photos.
✓
MEDIUM What is MVC architecture? ›
Best Answer
MVC (Model-View-Controller) separates concerns:
Model — handles data and business logic
View — UI presentation layer
Controller — receives user input, updates Model, selects View

Example in Spring Boot: @Repository (Model), Thymeleaf templates (View), @Controller classes (Controller).
✓
MEDIUM What is JWT? Why is it used? (If on resume) ›
Best Answer
JWT (JSON Web Token) is a compact, self-contained token used for secure information transmission between parties. Structure: Header.Payload.Signature.

Why use it: Stateless authentication — server doesn't need to store session. User logs in → gets JWT → sends JWT with every request → server verifies signature.
💡 If JWT is in your resume, expect deep questions. Know the three parts and the flow.
✓

HTML5 introduces semantic tags: <article>, <section>, <nav>, <header>, <footer>, <main>, <aside>. Also: <video>, <audio>, <canvas>, <datalist>. Native support for multimedia without plugins, geolocation API, local storage, WebSockets.

AI/ML & Modern Tech
TRENDING — AI questions now appear in almost every interview, all roles.
0/0 done
✓
HIGH What is Generative AI? What is Agentic AI? ›
Best Answer
Generative AI: AI that generates new content (text, images, code, audio) by learning patterns from training data. Examples: ChatGPT (text), DALL-E (images), Copilot (code).

Agentic AI: AI systems that autonomously plan and execute multi-step tasks using tools (search, APIs, code execution) to achieve goals without human intervention per step. They observe, reason, act, then re-evaluate.
✓
HIGH If AI can do your work, why should we hire you? ›
Best Answer
Great question to answer with confidence:

"AI is a powerful tool, not a replacement for human judgment. I see myself as someone who amplifies what AI can do — I understand the problem space, guide the AI effectively, verify its outputs, and apply domain knowledge that AI lacks. I use AI tools actively in my workflow, which makes me more productive than someone who doesn't. The combination of my technical skills and AI proficiency is what makes me valuable — not one or the other."
💡 Multiple panelists asked this. Show you USE AI tools, not fear them.
✓
HIGH What is tokenization in the context of AI? ›
Best Answer
Tokenization is the process of breaking text into smaller units called tokens (words, subwords, or characters) that an LLM can process. The model converts these tokens into numerical embeddings.

Example: "unhappiness" → ["un", "happiness"] or ["un", "happy", "ness"]. GPT-4 uses ~100k token vocabulary. Token limits define how much text a model can process at once.
✓
MEDIUM What is RAG (Retrieval-Augmented Generation)? ›
Best Answer
RAG combines an LLM with a retrieval system. Instead of relying only on training knowledge, the model retrieves relevant documents from a knowledge base at query time, then uses them as context to generate a more accurate, up-to-date answer.

Flow: User query → Vector search in DB → Retrieve relevant chunks → Pass to LLM as context → Generate grounded response.
✓
MEDIUM Supervised vs Unsupervised learning ›
Best Answer
Supervised: Model trains on labelled data (input-output pairs). Examples: classification (spam filter), regression (house price prediction).
Unsupervised: Model finds patterns in unlabelled data. Examples: clustering (customer segmentation), dimensionality reduction (PCA).
Reinforcement Learning: Agent learns by trial and error via rewards/penalties (chess, robotics).
✓
MEDIUM What is overfitting and underfitting? ›
Best Answer
Overfitting: Model memorizes training data, performs poorly on new data. Too complex. Fix: more data, regularization (L1/L2), dropout, cross-validation.
Underfitting: Model is too simple, poor performance even on training data. Fix: more features, more complex model, more training.
✓
MEDIUM What are weights in a neural network model? ›
Best Answer
Weights are the learnable parameters of a neural network — numerical values that determine the strength of connections between neurons. During training (backpropagation), weights are updated to minimize the loss function. An LLM like GPT-4 has billions of weights that collectively encode learned patterns from training data.
✓
MEDIUM What is rate limiting in APIs? ›
Best Answer
Rate limiting restricts the number of API requests a client can make in a given time window. Prevents abuse, ensures fair usage, protects server resources.
Example: Twitter API: 300 requests per 15 minutes. Common strategies: token bucket, sliding window, fixed window. Response: HTTP 429 Too Many Requests.

HIGH What is SDLC? What are Agile and iterative approaches? ›
Best Answer
SDLC (Software Development Life Cycle): Planning → Requirements → Design → Implementation → Testing → Deployment → Maintenance.

Waterfall: Linear, sequential. Each phase complete before next. Good for fixed-requirement projects.

Agile: Iterative cycles (sprints, typically 2 weeks). Deliver working software frequently, adapt to changing requirements. Daily standups, sprint reviews, retrospectives.

Agile vs Iterative: Agile is a specific agile methodology. Iterative is the broader concept of building in cycles.
